OverTheWire: Bandit Writeup

Bandit is the best starting point for anyone new to wargames. It teaches you the Linux command line through increasingly tricky challenges. Here’s my walkthrough.

bandit0#

This one is easy — the password is in the readme file.

bandit1#

To read files with special characters as the name, prepend ./:

cat ./-
# output: 263JGJPfgU6LtdEvgfWU1XP5yac29mFx

bandit2#

To read files with spaces, quote the filename:

cat 'spaces in this filename'
# output: MNk8KNH3Usiio41PRUEoDFPqfxLPlSmx

bandit3#

Use ls -la to see hidden files and directories:

ls -la inhere/

bandit4#

The inhere directory has many files. Use grep with a regex to find the human-readable one:

cd inhere
grep '[a-zA-Z0-9]' .
# -file07: 4oQYVPkxZOOEOO5pTW81FB8j8lxXGUQw

bandit5#

Find a file that is human-readable, 1033 bytes, and not executable:

find . -type f -size 1033c 2>/dev/null | xargs cat
# output: HWasnPhtq9AVKe0dmk45nxy20cvUa6EG

bandit6#

The file is owned by user bandit7, group bandit6, and is 33 bytes:

find / -user bandit7 -group bandit6 -size 33c 2>/dev/null | xargs cat
# output: morbNTDkSW6jIlUc0ymOdMaLnOlFVAaj

bandit7#

The password is next to the word “millionth” in data.txt:

grep millionth data.txt
# output: millionth    dfwvzFQi4mU0wfNbFOe9RoWskMLg7eEc

bandit8#

The password occurs only once in data.txt:

cat data.txt | sort | uniq -c | awk '{if ($1 < 10) print $0}'
# output: 1 4CKMh1JI91bUIZZPXDqGanal4xvAg0JM

bandit9#

The password is in data.txt, preceded by = characters:

strings data.txt | awk '{print $2}' | grep -E '.{32,}'
# output: FGUW5ilLVJrxX9kMYMmlN4MgbpfMiqey

bandit10#

The file is base64 encoded:

cat data.txt | base64 -d
# output: The password is dtR173fZKb0RRsDFSGsg2RWnpNVj3qRr

bandit11#

All letters have been rotated by 13 positions (ROT13). Decode with tr:

cat data.txt | tr 'A-Za-z' 'N-ZA-Mn-za-m'
# output: The password is 7x16WNeHIi5YkIhWsfFIqoognUTyj9Q4

bandit12#

The file is a hexdump of a repeatedly compressed file. Convert to binary, then iteratively decompress:

xxd -r data.txt > binary
mv binary binary.gz
gunzip binary.gz
# Check file type, rename, decompress... repeat
# Eventually: The password is FO5dwFsc0cbaIiH0h8J2eUks2vdTDwAn

The key lesson: always check the file type with file before attempting decompression.

bandit13#

This level gives you a private SSH key instead of a password:

ssh -i sshkey.private bandit13@bandit.labs.overthewire.org -p 2220
cat /etc/bandit_pass/bandit14
# output: MU4VWeTyJk8ROof1qqmcBPaLh7lDCPvS

bandit14#

Submit the current password to port 30000 on localhost using netcat:

nc localhost 30000
MU4VWeTyJk8ROof1qqmcBPaLh7lDCPvS
# output: Correct! 8xCjnmgoKbGLhHFAZlGE5Tmu4M2tKJQo

bandit15#

Submit the password to port 30001 using SSL/TLS:

openssl s_client -connect localhost:30001
# Paste the password to get the next one

bandit16#

Find the SSL-speaking port in range 31000-32000:

ss -tulnp | grep LISTEN
# Port 31790 stands out
ncat --ssl localhost 31790
# Submit password to receive SSH private key for next level

bandit17#

Find the only line that differs between passwords.old and passwords.new:

diff passwords.old passwords.new

OverTheWire Bandit is an excellent foundation for Linux command-line skills. If you’re new to security, start here.
